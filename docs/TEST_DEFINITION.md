# テスト仕様書 — 営業日報システム

## 1. テスト方針

### 目的

本書は営業日報システムの品質を担保するためのテスト仕様を定める。機能要件・権限制御・バリデーション・画面遷移を網羅的に検証する。

### テストレベル

| レベル | 説明 |
| --- | --- |
| 機能テスト | 各画面・機能が仕様通りに動作することを確認する |
| 権限テスト | ロール（SALES / MANAGER）による表示・操作制御を確認する |
| バリデーションテスト | 入力値チェックが正しく機能することを確認する |
| シナリオテスト | 業務フロー全体を通した動作を確認する |

### テスト環境

| 項目 | 内容 |
| --- | --- |
| テスト対象 URL | `http://localhost:3000`（ローカル環境） |
| ブラウザ | Google Chrome（最新版）、Safari（最新版） |
| テストデータ | 本書「テストデータ定義」に記載のデータを使用する |
